# Product Requirements Docs

These are lightweight PRD-style docs derived from the current roadmap and planning material.

They are intentionally high level: product problem, target outcome, scope, and dependencies. Engineering specifics belong in `docs/guides/` and `docs/reference/`.

Promotion rule:

1. Keep PRDs focused on user-facing outcomes and stable product scope.
2. Create or update a PRD when work is milestone-level and bundles multiple related implementation items.
3. Keep implementation detail in developer docs or issues, not in product PRDs.
